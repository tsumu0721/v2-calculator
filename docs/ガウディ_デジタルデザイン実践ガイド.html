Rhinoceros × Grasshopper × Claude Code — 実践ガイド

吊るした鎖を、
ひっくり返すところから構造解析まで。

ガウディはサグラダ・ファミリアの形を、鎖を吊るして重力に描かせ、それを上下反転して見つけた。この「逆さ吊り実験」は、Grasshopperの物理演算プラグイン Kangaroo でそのまま再現できる。本ガイドは、Rhino/Grasshopper初心者が Claude Code を相棒に、パラメトリックモデリングからカテナリーのフォームファインディング、Karamba3D による構造解析までを一気通貫で進めるための地図である。

OVERVIEW

全体像 — 3つの道具の役割分担

最初に押さえるべきは「どの道具が何を担当するか」である。役割が分かれば、学ぶ順番も自然に決まる。

道具役割初心者が学ぶこと
Rhinoceros 83Dモデリングの土台。ジオメトリの表示・編集・保存(.3dm)画面操作、レイヤー、基本コマンド(Line, Curve, Loft)だけで十分
GrasshopperRhinoに内蔵のビジュアルプログラミング環境。「手順」を組んで形を生成するコンポーネントの接続、スライダー、そして最重要のデータツリー
Kangaroo 2GH内の物理演算。逆さ吊り実験(フォームファインディング)を担当Anchor・Load・Length(ばね)・Solverの4点セット
Karamba3DGH内の構造解析(FEM)。梁・シェルの応力・変形・座屈を計算「モデル組立→解析→結果表示」の定型パイプライン
Claude Codeあなたの側に立つエンジニア。GH用Pythonコードの生成、データ処理、レポート化、質問相手プロジェクトフォルダの作り方と、良い依頼の仕方
核心となる分業: 形の定義と可視化はGrasshopperのキャンバス上で、ロジックが複雑になる部分(数式、データ整形、最適化ループ、レポート生成)はClaude Codeに書かせたPythonで、と分けると初心者でも破綻しない。

PHASE 0 — 環境構築(1日)

環境構築

インストールするもの

  1. Rhino 8(90日評価版あり)。Grasshopperは同梱、追加インストール不要。Rhino 8からPython 3(CPython)がScriptEditorとGHPython双方で使え、numpy等のpipパッケージも導入できる。
  2. Kangaroo 2 — Rhino 8に標準同梱(GHの「Kangaroo2」タブを確認)。
  3. Karamba3D — food4rhino.comから入手。無償のtrial/教育ライセンスで学習用途は足りる。
  4. Claude Code — ターミナルから npm install -g @anthropic-ai/claude-code。プロジェクトフォルダで claude を実行して開始。
  5. (任意)Rhino MCP系プラグイン — ClaudeからRhinoを直接操作するコミュニティ製MCPサーバーが複数ある。導入すれば「Claudeに話しかけるとRhinoに形が生える」体験ができるが、必須ではない。まずはコード貼り付け方式で始めるのが確実。

Claude Codeとの接続方式は3つ

方式仕組み向き
① 貼り付け方式
(推奨スタート)		Claude CodeにGHPython用コードを書かせ、GHのPythonコンポーネントに貼り付ける初心者。確実で、コードが手元に残り学習になる
② ファイル連携方式GHがCSV/JSONを書き出し→Claude Codeのスクリプトが処理→結果をGHが読み戻す解析結果の集計・レポート化。本ガイドの主軸
③ MCP直結方式MCPサーバー経由でClaudeがRhino/GHを直接操作慣れてから。環境依存のトラブルがあり得る
初日のゴール: Rhinoを起動→Grasshopperとコマンド入力→キャンバスにSliderとCircleを置いて半径が動くこと。そしてClaude Codeに「GHPythonで円を10個並べるコードを書いて」と頼み、動かすこと。ここまでで環境は合格。

PHASE 1 — Grasshopperの考え方(1〜2週)

基礎:データツリーを制する者がGHを制す

Grasshopperの挫折ポイントは操作ではなくデータツリー(データの階層構造)である。ここだけは腰を据えて理解したい。

データツリーとは

GHのデータは「パス付きのリストの集まり」で流れる。たとえば柱を5本、各柱に10個の点を作ると、データは {0;0}〜{0;4} という5つの枝(branch)に、各10個の要素が入った形になる。

{0;0} → [pt0, pt1, ... pt9]   # 1本目の柱の点列
{0;1} → [pt0, pt1, ... pt9]   # 2本目の柱の点列
...
{0;4} → [pt0, pt1, ... pt9]   # 5本目

コンポーネントは「同じ枝同士」を対応させて計算する。思った通りに動かない時の原因の9割は、枝の構造の不一致である。Flatten(全部1列に潰す)、Graft(要素を1個ずつ枝に分ける)、Simplify(パスの余分な階層を消す)の3操作をまず覚える。

Claude Codeの使い方: ツリーでハマったら、パネルに表示されたパス構造(例: {0;0;1} N=12)をそのままClaudeに貼って「この2入力を対応させるにはFlatten/Graftのどれを何に掛ける?」と聞く。データツリーの質問相手としてAIは極めて有効。

この段階で作る練習課題

  1. サイン波で高さが変わる柱の列(Series → 数式 → Move)
  2. 2本の曲線間のLoftで壁を作り、Contourで水平スライス
  3. GHPythonコンポーネントで同じことをコードで再現(Claudeに書かせて読む)

PHASE 2 — ガウディの幾何学(1〜2週)

ガウディの形は「定規で描ける曲面」でできている

サグラダ・ファミリアの複雑に見える形は、実は少数の線織面(ルールドサーフェス)の組み合わせでできている。直線を動かして生成できる曲面なので、パラメトリックモデリングと極めて相性が良い。

幾何学使われている場所GHでの作り方
カテナリー(懸垂線)アーチ、ヴォールトの断面形状y = a·cosh(x/a) を数式 or GHPythonで点列生成 → Interpolate。またはKangarooで動的に生成
双曲面(Hyperboloid)天井の採光口、柱の接合部上下2つの円をDivideし、位相をずらして点同士を直線で結ぶ(Line) → Loft
双曲放物面(HP曲面)屋根、壁面ねじれの位置にある2直線をDivide → 対応点をLine → Loft
螺旋面(Helicoid)階段、柱の表面の捻れ直線を回転+上昇させながらコピー → Loft
樹状柱(二重回転柱)身廊の柱。星形断面が上昇しつつ逆向きに二重回転し、上部で枝分かれ星形多角形断面を、高さに応じて正逆2方向に回転させた断面群 → Loft。分岐はVectorで傾けた枝を複製

Claude Codeへの依頼例(GHPython)

# Claude Codeへのプロンプト例:
「Rhino 8のGHPython(Python 3)で、双曲面の柱を生成する
コンポーネントを書いて。入力: 下円半径r1、上円半径r2、高さh、
分割数n、ねじれ角theta(度)。出力: 母線のLineリストとLoft面。
rhinoscriptsyntaxではなくRhino.Geometryを使って。」

返ってきたコードをGHPythonコンポーネントに貼り、入力端子名をコードの引数に合わせるだけで動く。エラーが出たら全文をそのままClaudeに貼る。この往復自体が最良の教材になる。

PHASE 3 — フォームファインディング(2週)

逆さ吊り実験を Kangaroo で再現する

ガウディは模型の鎖に小袋の重りを吊るし、純引張のつり合い形状を作り、写真を上下反転して純圧縮のアーチ・ヴォールトを得た。Kangaroo 2はこれを数値計算(動的緩和法)で行う。

Kangarooの基本レシピ(単一アーチ)

GH1. 鎖のモデル化2点間にLineを引き、Divideで20分割 → 隣接点間の線分リストを作る(これが鎖のリンク)
Kangaroo2. ゴール設定Length(各リンクをばね化、目標長=元長×0.9で少し縮ませる)+ Load(全節点に-Z方向の重力)+ Anchor(両端点を固定)
Kangaroo3. Solver実行ZombieSolverにゴールを接続 → つり合ったカテナリー形状の点列が出力される
GH4. 反転Mirror(XY平面)で上下反転 → 圧縮アーチの完成。Interpolateで曲線化

これをネットワークに拡張すると、複数アーチが連なるヴォールト(吊り模型そのもの)になる。メッシュのエッジをリンクに、交点に重りを掛ければよい。

数式との突き合わせ: Kangarooの出力点列をCSVに書き出し、Claude Codeに「この点列をy=a·cosh(x/a)でフィッティングしてaを推定し、残差をグラフ化して」と頼む。物理演算と理論式が一致することを自分の目で確かめると、理解が一段深くなる。

PHASE 4 — 構造解析(2〜3週)

Karamba3D で「その形は立つのか」を検証する

Karamba3DはGH内で完結する有限要素解析(FEM)プラグイン。パイプラインは常にこの定型で、覚えるのは一度だけでよい。

GH1. ジオメトリ → 要素化アーチ曲線をShatterで分割 → LineToBeam(梁要素)。面はMeshToShell(シェル要素)。ここで要素ID(名前)を付けておくと後の集計が楽
Karamba2. 断面と材料CrossSection(円形断面 φ300 など)+ Material(石材相当なら E≈50GPa・圧縮強め・引張弱めの特性、鋼ならS235等のライブラリ値)
Karamba3. 支持と荷重Support(アーチ脚部を固定: Tx,Ty,Tz,Rx,Ry,Rz)+ Load(自重はGravity一発。加えて点荷重・ライン荷重を設定)
Karamba4. 組立と解析AssembleModel → AnalyzeThI(一次解析)。大スパンならThII(幾何非線形)や座屈解析(BucklingModes)へ
Karamba5. 結果の取得ModelView/BeamView(変形図・応力色分け)、Utilization(検定比)、NodalDisplacements、BeamForces(N, V, M)
Claude Code6. 書き出しと帳票化要素ID・応力・変位・検定比をCSVへ書き出し → Pythonで集計・グラフ化 → 性能確認書(docx/HTML)を自動生成

カテナリーの威力を数値で見る実験

同スパン・同断面で ①半円アーチ ②放物線アーチ ③カテナリーアーチ の3本を並べて自重解析し、曲げモーメントMの最大値を比較する。カテナリーだけMがほぼゼロ(軸力のみ)になる。ガウディが鎖に形を聞いた理由が、数字で腑に落ちる瞬間である。

正直な注意: Karambaの標準は弾性の梁・シェル解析であり、実際の組積造(石積み)の挙動(目地の開き、引張を持てない性質)の厳密な再現は上級テーマ(リミットアナリシス等)になる。学習段階では「圧縮主体の形状ほど曲げが消える」ことの確認、つまり形状の良否比較に使うのが正しいスタンス。

DATA — プロジェクトのデータ構成

データ構成 — Claude Codeと共有するフォルダ設計

Claude Codeはフォルダの中身を読んで働く。GHとの受け渡しファイルを規約化しておくと、依頼が一言で済むようになる。

sagrada-study/
├── CLAUDE.md              # Claude Codeへの常設指示(規約・単位系・目的)
├── gh/
│   ├── 01_geometry.gh     # 幾何生成(双曲面柱・カテナリー)
│   ├── 02_formfinding.gh  # Kangaroo逆さ吊り
│   └── 03_analysis.gh     # Karambaパイプライン
├── rhino/
│   └── model.3dm          # ベースのRhinoファイル
├── exchange/              # GH ⇄ Python の受け渡し(規約の心臓部)
│   ├── params.json        # 入力パラメータ(スパン、断面径、荷重…)
│   ├── nodes.csv          # id, x, y, z
│   ├── elements.csv       # id, node_i, node_j, section, material
│   └── results.csv        # element_id, N, V, M, util, disp_max
├── scripts/               # Claude Codeが書くPython
│   ├── fit_catenary.py    # カテナリーフィッティング
│   ├── summarize.py       # 結果集計・グラフ生成
│   └── make_report.py     # 性能確認書(docx/HTML)生成
├── output/
│   ├── figs/              # グラフ・キャプチャ
│   └── report.docx        # 自動生成レポート
└── notes/                 # 学習メモ(Claudeに要約させると復習が楽)

受け渡しデータの定義例

# params.json — GHが読み込み、スライダー既定値を上書きする
{
  "span_m": 12.0,
  "rise_m": 8.0,
  "segments": 24,
  "section": {"type": "circle", "d_mm": 300},
  "material": "stone_E50GPa",
  "loads": [{"kind": "gravity", "factor": 1.0}]
}
CLAUDE.mdに書くべきこと: ①単位系(長さm・力kN・応力MPaに統一) ②exchange/ のスキーマ ③「results.csvの列名は変えない」等の禁止事項 ④座標系(Z上向き)。これだけでClaude Codeの出力が安定する。あなたのCEOエージェント構成を流用するなら、scripts/ 配下の各処理をサブエージェントのタスクとして割り振る形にそのまま載る。

CAPSTONE — 集大成(3〜4週)

ミニ・サグラダ:身廊1ベイの生成と解析

実物の完全再現ではなく、身廊1スパン(1ベイ)の簡略モデルを自作パラメータで組む。これが初心者の到達点として現実的で、かつ全技術が入る。

  1. 柱: Phase 2の樹状柱(星形断面の二重回転)を4本。上部でVectorを傾けた枝分かれを付ける
  2. ヴォールト: 柱頂部の枝先をAnchorにしたKangarooネットで逆さ吊り → 反転して圧縮ヴォールト面を生成
  3. 解析: 柱=梁要素、ヴォールト=シェル要素としてKarambaへ。自重+屋根荷重で変形・検定比を確認
  4. 最適化: Galapagos(GH標準の遺伝的アルゴリズム)で「重量最小かつ検定比<1」となる断面径・ライズ比を探索。Phase 3で作ったグリッドサーチ的発想のGH版
  5. 帳票: results.csvをClaude Codeに渡し、比較表・応力図・結論をまとめたレポートを自動生成

つまずいた時の鉄則

  • エラー文・パネルのツリー表示・スクリーンショットの説明文をそのままClaudeに貼る。要約しない方が早い
  • GHファイルが巨大化したらGroup+名前付けで区画整理。「このGroupは何をしている?」とClaudeに説明させて検算する

  • 単位系の混在(mmとm)は構造解析で必ず事故る。CLAUDE.mdの規約を正とする
