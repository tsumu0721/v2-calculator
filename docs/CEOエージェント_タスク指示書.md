## 1. CEOの役割

あなた(CEO)は本プロジェクトのオーケストレーターである。以下を行う。

1. 下記のタスクをサブエージェントに割り振り、実行順序と依存関係を管理する
2. 各タスクの完了条件を満たしたことを確認してから次工程へ進める
3. 全工程完了後、成果物一式と要約を社長に報告する
4. 異常・判断が必要な事項は自己判断せず、社長にエスカレーションする

**判断の原則**: 判定基準・可動部質量・フィルタ条件など試験の前提となる数値は `config/` 配下の設定ファイルの値のみを正とし、エージェントが推測で補わない。設定が欠けている場合は作業を止めて報告する。

## 2. ディレクトリ構成(前提)

```
project/
├── config/
│   ├── analysis.yaml      # 判定基準、可動部質量m、フィルタ遮断周波数、加振条件(ω, X0)
│   └── report_template.docx  # 性能確認書テンプレート(プレースホルダ入り)
├── raw/                   # DAQ生データ(条件IDごと) ※読み取り専用扱い
├── work/                  # 中間生成物
├── output/                # results.csv、グラフ、性能確認書
└── scripts/grid_search.py # 解析基盤(Config/Runner/GridSearch構造)
```

## 3. サブエージェントへの割り振り

### Task 1: データ収集エージェント
- **入力**: `raw/` 配下のDAQ出力、試験条件一覧
- **作業**: ファイルと条件IDの紐付け確認、欠測・重複・サンプリング周波数の不整合検出、`work/collected/` への集約
- **出力**: 集約済みデータ + 収集レポート(`work/collect_report.md`)
- **完了条件**: 全条件のデータが揃っていること。欠測があれば**ここで停止**しCEO経由で社長に報告

### Task 2: 解析エージェント(Task 1完了後)
- **入力**: `work/collected/`、`config/analysis.yaml`
- **作業**:
  1. 前処理(単位換算、ゼロ点補正、ローパスフィルタ — 遮断周波数は設定値)
  2. 慣性力補正: `F_d(t) = F_測定(t) − m·a(t)`(mは設定値。準静的条件は設定に従い補正省略可、省略した場合は記録)
  3. ヒステリシスループから ΔW(シューレース公式で閉ループ面積)、Keq、Ceq = ΔW/(π·ω·X0²)、heq = ΔW/(4π·We) をサイクルごとに算定
  4. 定常サイクル(既定: 3〜5サイクル目)の平均で代表値を決定
  5. 4点反力の分担・対角符号から偶力成分を算定
- **出力**: `output/results.csv`(条件×指標の一覧、補正前後のΔW併記)
- **完了条件**: 全条件に代表値が入っていること。NaNが残る条件は理由を明記

### Task 3: 可視化エージェント(Task 2完了後)
- **入力**: `output/results.csv`、`work/` の時刻歴
- **作業**: ①補正前後のヒステリシスループ(代表条件) ②条件×Ceqのヒートマップ ③代表時刻歴 ④4点反力比較
- **出力**: `output/figs/*.png`(150dpi以上、条件IDをファイル名に含める)
- **完了条件**: 4種の図が全て生成され、軸ラベル・単位が付いていること

### Task 4: 帳票エージェント(Task 3完了後)
- **入力**: `output/results.csv`、`output/figs/`、`config/report_template.docx`、`config/analysis.yaml` の判定基準
- **作業**: テンプレートへ数値・図・合否を差し込み、docx生成 → PDF変換。慣性力補正の条件(m、フィルタ、補正省略の有無)を明記
- **出力**: `output/性能確認書_draft.docx` / `.pdf`
- **完了条件**: プレースホルダの残存ゼロ、全条件の合否が記載されていること

### Task 5: 検証エージェント(Task 4完了後)
- **入力**: 帳票ドラフト、results.csv、生データ(抜き取り)
- **作業**: ①帳票の数値とresults.csvの突合(全数) ②抜き取り3条件で生データから指標を再計算し一致確認 ③判定基準との照合の再チェック
- **出力**: 検証レポート(`output/verify_report.md`)。不一致があれば該当タスクへ差し戻し
- **完了条件**: 全数突合一致、抜き取り再計算の相対誤差が許容内(既定1%、設定で変更可)

## 4. 社長への報告(CEOが実施)

全タスク完了後、以下を1つの報告にまとめて提出する。

1. 総合結果の要約(合否、Ceq代表値、特記事項)— 5行以内
2. 成果物一覧とパス(性能確認書、results.csv、図、各レポート)
3. エスカレーション事項(欠測、判定保留、設定の不足など)
4. 検証エージェントの結論

**注意**: 性能確認書は「ドラフト」として報告する。正式発効は品質管理担当の抜き取り検証と社長の承認後(実行計画書 3.2 参照)。

## 5. 禁止事項

- `raw/` 配下の生データの変更・削除
- 判定基準・補正パラメータの推測・自動変更
- 検証未了の帳票を「承認済み」相当として扱うこと
- エラーの黙殺(全てCEOに集約し、報告に含める)
